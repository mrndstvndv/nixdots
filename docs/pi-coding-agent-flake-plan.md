# pi-coding-agent flake integration plan

## Goal

Make `modules/pi` ready to consume a separate `pi-coding-agent` flake, install the `pi` CLI from that flake, and generate `lastChangelogVersion` from the packaged pi version so Home Manager's immutable `settings.json` does not drift.

## Constraints

- `~/.pi/agent/settings.json` is generated by Nix, so pi cannot be trusted to persist `lastChangelogVersion` itself.
- The external flake does not exist in this repo yet, so nixdots should support it optionally without requiring the input today.
- `settings.json.packages` is for pi packages/extensions, not the pi CLI derivation.

## Plan

- [x] Add an optional `piAgent` special arg path through `flake.nix` so all Home Manager targets can consume a future external flake input without breaking current builds.
- [x] Update `modules/pi/default.nix` to:
  - [x] optionally install `piAgent.packages.${pkgs.system}.default` or `.pi`
  - [x] derive `lastChangelogVersion` from the installed pi package version when `piAgent` is present
  - [x] fail fast if a `piAgent` input is provided but exposes neither a package nor a version
  - [x] keep the existing local pi extensions package wiring untouched
- [x] Remove fallback `lastChangelogVersion` from `modules/pi/settings.json` so it only comes from the packaged pi version.
- [x] Document the new contract in `modules/pi/README.md`, including the expected external flake outputs.

## Expected external flake contract

The future external repo should expose at least:

```nix
{
  packages.${system}.default = ...; # or packages.${system}.pi
  apps.${system}.default = {
    type = "app";
    program = "${self.packages.${system}.default}/bin/pi";
  };
}
```

Recommended:

- derivation has a `version` attr
- optionally also expose `lib.version = "x.y.z"`

## Result

After these changes, nixdots will be ready for:

1. adding `piAgent.url = "github:<you>/pi-coding-agent-flake"` later
2. passing that input into the pi module automatically
3. keeping `lastChangelogVersion` in sync with the packaged pi CLI version
